## Requisitos

Lista los requisitos necesarios para el proyecto:

* Resolución del problema: En el proyecto se utilizan adecuadamente las instrucciones de programación y algoritmos tratado de no generar redundancias y optimizando el código
* Usabilidad. El diseño de la interfaz es agradable y fácil de utilizar, no requiere de mucho tiempo para su utilización.
* Funcionalidad: La aplicación de la propuesta funciona perfectamente al realizar cada una de las tareas y está debidamente validado..
* Documentación: Cuenta con documentación interna que aporte comentarios respecto al código para entender claramente la estructura del programa
* La presentación del proyecto incluye todos los requerimientos solicitados.

## Uso

* Cada quien debe clonar el proyecto en su compu.
* Cuando hagan sus cambios, tienen que hacer un commit para guardarlos, y luego un Pull request para mandarlo al proyecto (rama) principal.
* Ya con eso el Angel se encarga de que se integre a nuestro proyecto
* Cuando se haga un cambio cada quien debe acualizar el proyecto en su compu usando git pull.
